Reading: JavaScript-Novice to Ninja Chapter: 10 Testing and Debugging

An exception is an error that produces a return value that can then be used by the program to deal with the error.

A stack trace is a sequence of functions or method calls that lead to the point where the error occurred.

A warning can occur if there’s an error in the code that isn’t enough to cause the program to crash.

To use the strict mode you only need the following string to be added to the first line of a JavaScript file: 'use strict';
ES6 introduced JavaScript modules. These are selfcontained pieces of code that are in strict mode by default, so the 'use strict' declaration is not required.

Linting tools such as JS Lint,1 JS Hint,2 and ES Lint3 can be used to test the quality of JavaScript code, beyond simply using strict mode.
They are designed to highlight any sloppy programming practices or syntax errors, and will complain if certain style conventions are not followed, such as how code is indented.
They are useful for enforcing a programming style guide if we are working with a team.

Breakpoints are useful if we are debugging in a browswer, this halt the progress of the code and allow us to view the value of different variables at that point in the program.

Console object that provides a number of methods for logging information and debugging for example the famous console.log() statements

One of the most useful commands is the debugger keyword. This will create a breakpoint in your code that will pause the execution of the code and allow you to see where the program is currently up to. The only thing is we need to remember delete the this command when we finish all our code because if we don't the program will appear to freeze when people try to use it

We can create error objects, they can be created in the code using a constructor function:
There are seven more error objects used for specific errors: (Check the book for more specifications)
EvalError, RangeError, ReferenceError, SyntaxError, TypeError, URIError and InternalError.
All error objects have a number of properties, the properties that are generally safe to use are:
Name, message and stack.

It’s also possible to throw your own exceptions using the throw statement. This will allow for any problems in your code to be highlighted and dealt with, rather than lurk quietly in the background.

The prototype can be used to add any new properties and methods after the class has been declared. It should be used to define any properties that will remain the same for every instance of the class.

We can add properties to an object using the Object.defineperty() method and we can see this values with the Object.getOwnPropertyDescriptor() method.

We can use the get() method to return anything we like.

We can create a new object that is the exact copy from another with the same name.

A mixin is a way of adding properties and methods of some objects to another object without using inheritance.

A module is a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules.

Default exports refer to a single variable, function or class in a module that can be imported without having to be explicitly named. So it can be use it in another .js file.
