These are the two different index notation we can use to return the first form object:
const form = document.forms[0];
const form = document.getElementsByTagname('form')[0];

Instead of a numerical Index we can use the name attribute:
const form = document.forms.search;

This returns all the HTML collections of a form:
const [input,button] = form.elements;

The different form properties and methods:
The form.submit() method will submit the form automatically or it can be submitted manually by the user with a button or input element with a type attribute of submit.
The form.reset() method will reset all the form controls back to their initial values specified in the HTML. But this is sometimes poor in usability and not recommended.
The event.preventDefault() method can stop the form from being submitted in case there is an empty default value or other.

The JSON.stringify() can convert an object into a JSON string.

Input fields can be text, password, checkbox, radio, hidden, file and other inputs like color, telephone, number, etc.

There are different kinds of buttons: the most common the submit.

Object-oriented programming is often used to model representations of objects in the real world.
There are three main concepts in OOP: encapsulation, polymorphism, and inheritance.
Encapsulation: we can’t see how the object works because just the basic is exposed to the user.
Polymorphism: We can use different objects in the same process.
Inheritance: We can add new features to a new object from another previously created.
I’m taking a Java class, it’s my first time with it and it’s so interesting the similarities with both languages.

In a constructor function the keyword THIS is used to represent the object name.

The Object.getPrototypeOf() method can takes an object as a parameter.

The prototype can be used to add any new properties and methods after the class has been declared. It should be used to define any properties that will remain the same for every instance of the class.

We can add properties to an object using the Object.defineProperty() method and we can see this values with the Object.getOwnPropertyDescriptor() method.

We can use the get() method to return anything we like.

We can create a new object that is the exact copy from another with the same name.

A mixin is a way of adding properties and methods of some objects to another object without using inheritance.

A module is a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules.

Default exports refer to a single variable, function or class in a module that can be imported without having to be explicitly named. So it can be use it in another .js file.
